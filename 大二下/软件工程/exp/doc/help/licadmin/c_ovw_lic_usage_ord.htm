Understanding license usage order

IBM Rational products request licenses in a specific order. If you have a combination of node-locked and floating license keys in your environment, you may want to change the order.

An IBM product that uses Rational Common Licensing (powered by FLEXlm software) requests licenses in this order, by default:

  1. Node-locked before a floating license. An IBM Rational product requests a node-locked license before it requests a floating license regardless of whether it is a Suite or point-product license.
  2. A point-product license before a Suite license. For example, Rational Purify is sold as part of a Rational Suite and as a stand-alone product. Among a group of node-locked and floating licenses, Rational Purify will request a Rational Purify license before it requests the Rational Suite license.
  3. Suite license of the last Rational Suite installed.

To change the order on your client, use the License Usage Mapper in the License Key Administrator. To change the order for Web clients, use the License Usage Mapper on the Web server.

The license usage order that you set on your computer is tied to your user ID. If different users log in to your computer with their own user IDs, the license usage order resets to the default order. The new users can set the license usage order for their user IDs. Their settings do not affect your settings.

The IBM Rational license server does not change license usage order for all clients and all users.
