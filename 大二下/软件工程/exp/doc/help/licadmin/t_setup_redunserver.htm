Setting up redundant license servers

You can configure a redundant license server environment.

To understand why you may want to set up a redundant server environment, see Understanding redundant license servers.

To set up redundant license servers:

  1. Install the license server software on each server computer. For instructions, see the IBM Rational License Management Guide in the Rational common directory.
    Note: Do not implement servers on a mix of operating systems (Windows, Linux, and the UNIX system). Use a redundant environment of all servers on the same operating system.
  2. Make sure that the three servers operate on the same TCP/IP port.
  3. Make sure that other processes are not sharing the TCP/IP port.
  4. Enter or import the license keys on each license server.
  5. To configure named-user floating licenses, edit the options file.
  6. Start each license server.
  7. Enter port values (optional).
  8. Give users the license server names (in the correct order: Primary, Secondary, and Tertiary) to enter into their License Key Administrator. This step is not necessary if the server names were entered into a site defaults file in a network release area. For more information about release areas, read your IBM Rational product installation guide.
